FOCUS designs coils for a finite-beta equilibrium¶

Unlike vacuum configurations, a finite-beta equilibrium usually has non-zero plasma currents. This implies the plasma currents will contribute to the normal field on the plasma boundary. The basic logic is to calculate the normal field from plasma currents and save it as an input to FOCUS. FOCUS will design coils satisfying the boundary conditions.

Here we are going to use the BNORM code calculating the required normal field from plasma currents read from a VMEC output. The most important part is to make sure you are using the correct sign and normalization. There is a detailed discussion about the coodinate systems and normalizations in several different codes, please check Coil_design_codes_benchmark.

We use the NCSX C09R00 configuration as a reference. The files can be found in examples/ncsx_c09r00.

  1. Prepare FOCUS input Here we use the function vmec2focus in the python package CoilPy.
    from coilpy import vmec2focus
 vmec2focus('wout_c09r00_fb.nc', focus_file='c09r00.boundary', bnorm_file='bnorm.c09r00_fb')
  2. Run FOCUS with actual coils and generate a MGRID file 
     mpiexec xfocus actual
  3. Run free-boundary VMEC with FOCUS written MGRID file 
     mpiexec xvmec2000 input.c09r00_focus

Note:

  1. Right now, FOCUS cannot constrain the total poloildal current. This might cause trouble when doing equilibrium reconstruction. It is suggested to fix the coil currents.
  2. If you are using FAMUS, you have to specify flipsign=True in the arguments because FAMUS adds the input Bn.
